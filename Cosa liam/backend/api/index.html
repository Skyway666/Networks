Incident record

incident

Retrieve a group of incidents

GET /incident

Incidents will be paginated to 20 incidents by default.

You can specify further pages with the ?page parameter. You can also set a custom page size up to 100 with the ?perpage parameter.

Incidents can also be sorted supplying ?sort and ?sortdir parameters.

Lastly, incidents can be filtered supplying the ?search parameter, it will limit the results to those whose employee, location or injury type match the supplied string.

page: integer
in query

Specifies the page that will be returned.

perpage: integer 1 ≤ x ≤ 100 20
in query

Specifies the number of incidents per page.

sort: string id, datecreated, employee, location, injurytype
in query

Specifies the field used for sorting.

sortdir: string asc, desc
in query

Specifies the sorting order. Use " asc" for ascending order or " desc" for descending order.

search: string
in query

This parameter will limit the incidents returned to those whose employee, location or injury type match the supplied string.
